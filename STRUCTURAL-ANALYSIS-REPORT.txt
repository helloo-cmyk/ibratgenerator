================================================================================
2. GENERATOR LOCATION
================================================================================

Where the generator lives:
- UI shell: src/components/BratGenerator.tsx — renders full widget DOM, styles, triggers init
- Logic: src/components/brat-generator-init.ts — wires DOM, canvas, event listeners

Which pages load it: page.tsx (home), brat-maker, brat-creator, brat-text-generator, brat-name-generator

Call chain:
  page.tsx
    -> BratGeneratorLazy (dynamic, ssr: false)
      -> BratGenerator
        -> useEffect: loadScript(JSZip), loadScript(FileSaver)
        -> initBratGenerator() from brat-generator-init.ts
          -> getElementById for canvas and controls
          -> state, historyStack, redoStack
          -> draw(), requestDraw(), setCanvasSize()
          -> addEventListener on canvas and window
          -> return cleanup

================================================================================
3. GENERATOR USAGE ACROSS PAGES
================================================================================

All 5 generator routes: /, /brat-maker/, /brat-creator/, /brat-text-generator/, /brat-name-generator/
- Generator loaded: Yes
- State: Independent per page (no sharing)
- Props: None. <BratGeneratorLazy /> with no props.

================================================================================
4. GENERATOR INITIALIZATION FLOW
================================================================================

1. Page loads -> Next.js serves route
2. BratGeneratorLazy renders -> dynamic import BratGenerator (ssr: false), shows "Loading generator..."
3. BratGenerator mounts -> renders #brat-embed-root, controls, canvas
4. useEffect: load JSZip and FileSaver from CDN, then initBratGenerator()
5. initBratGenerator: resolve DOM by ID, create state, setCanvasSize(), switchTab("text"), attach listeners
6. Event listeners: canvas (pointerdown, dblclick, wheel), window (pointermove, pointerup, keydown, resize), controls (input/change/click)

Important functions: initBratGenerator(), requestDraw(), draw(), setCanvasSize(), undo(), redo(), pushHistory()
No applyTemplate() in codebase.

================================================================================
9. FINAL SUMMARY
================================================================================

Main generator page: Homepage (/) plus brat-maker, brat-creator, brat-text-generator, brat-name-generator
Pages that reuse generator: Same 5 routes, same component, no props
Structure: React mounts DOM, brat-generator-init.ts holds state and behavior in closure

Architectural limitations for templates:
- No template/preset API or props for initial theme
- Single global DOM, fixed IDs
- No shared state across routes
- Color presets only via 4 pills; templates would need props or URL params
