CORE WEB VITALS AUDIT
=======================

CHECK 1A — LCP (Largest Contentful Paint)
- LCP Element (Homepage): H1 "Brat Generator" or the BratGeneratorLazy container.
- LCP Element (Tool Pages): H1 tool title.
- Analysis:
  * H1 is the first visible text on all pages.
  * No large images are above the fold.
  * BratGeneratorLazy is correctly wrapped in minHeight: 520px, preventing layout shifts.
  * next/dynamic is used with ssr: false for the generator.
  * Issue: BratGenerator is a large JS chunk that may delay interactivity.
  * Layout: Fonts load with display: swap. next/script is used with afterInteractive.

CHECK 1B — CLS (Cumulative Layout Shift)
- Sources Found:
  * PartnerBadges.tsx: Uses <img> without explicit width/height attributes (Style only). [MEDIUM Risk]
  * BratGeneratorLazy: Reserved space is good, but potential for minor shifts exists. [LOW Risk]
  * ExampleGallery: Uses aspect-square wrappers and next/image with dimensions. [SAFE]
  * CookieConsentBanner: Potential to push content if not fixed-positioned. [LOW Risk]

CHECK 1C — INP (Interaction to Next Paint)
- Bottlenecks:
  * BratGenerator.tsx: Heavy canvas operations and many event listeners. Synchronous drawing on keystrokes.
  * brat-generator-init.ts: Deep cloning of state on every history snapshot (up to 50) could cause lag.
  * FAQ Accordion: uses max-height transitions which can be heavy.

CHECK 1D — FID / TBT (Total Blocking Time)
- Analysis:
  * Third-party scripts (GA, Adsense, Clarity) are loaded with afterInteractive.
  * Build Output: All pages are pre-rendered as Static (o).
  * Bundle Size: Shared chunks are optimized, but BratGenerator remains the largest client-side component.
